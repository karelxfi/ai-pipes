## What Each Indexer Includes

Every example is self-contained and follows the same structure:

```
evm/001-aave-v3/
  src/index.ts          # Indexer source (Pipes SDK)
  dashboard/            # ECharts dashboard + screenshot
  validate.ts           # 3-phase validation (structural + Portal cross-ref + tx spot-checks)
  contracts.json        # Verified contract addresses (fed to contracts-registry)
  PROMPT.md             # Exact prompt used to generate this indexer
  OUTPUT.md             # Decisions made, issues encountered
  META.json             # Date, SDK version, model, angle
  README.md             # Verification report, run instructions, sample query
```

### Validation

Every indexer is validated against [SQD Portal](https://portal.sqd.dev) — not just structurally checked:

1. **Structural** — row count, schema, timestamp range, non-empty fields
2. **Portal cross-reference** — independent event count comparison (within 5% tolerance)
3. **Transaction spot-checks** — 2-3 txs verified field-by-field against Portal data

## How It Works

1. Pick a protocol from `protocols.json`
2. Claude Code researches the protocol and picks a compelling angle
3. The `pipes-new-indexer` skill scaffolds a working indexer
4. Dashboard, validation script, and metadata are generated
5. Indexer runs locally, data is validated against Portal, screenshot is captured
6. Verified contract addresses are pushed to [contracts-registry-llm](https://github.com/karelxfi/contracts-registry-llm)

## Quick Start

Pick any example and run:

```bash
cd evm/001-aave-v3
docker compose up -d
npm install
npm start
```

Open `dashboard/index.html` in your browser to see the charts.
