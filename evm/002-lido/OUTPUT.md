# Output — Lido stETH Rebase Tracker

## What was done

1. **Scaffolded indexer** using `@iankressin/pipes-cli` with the `custom` template, targeting the Lido stETH contract on Ethereum mainnet (`0xae7ab96520DE3A18E5e111B5EaAb095312D7fE84`) from block 17,000,000 (~May 2023, post-Shanghai upgrade).

2. **Fixed proxy contract ABI** — The Lido contract uses Aragon's AppProxyUpgradeable, which neither the CLI nor `@subsquid/evm-typegen` can auto-resolve. The generated ABI only had `ProxyDeposit`. Manually wrote the contract types file with the `TokenRebased` event definition, computing topic0 via keccak256.

3. **Configured dedicated ClickHouse database** — Changed from default `pipes` to `lido` to avoid sync table conflicts with other indexers.

4. **Created dashboard** — Four-chart ECharts layout:
   - stETH/ETH exchange rate over time (area/line chart)
   - Total pooled ETH / TVL growth (area chart)
   - Rebase frequency per week (bar chart)
   - Fee shares minted per day (bar chart)
   - Summary stats: total rebases, current exchange rate, total pooled ETH

5. **Created validation script** — Three-phase validation:
   - Phase 1: Structural checks (row count, schema, timestamps, exchange rate sanity)
   - Phase 2: Portal cross-reference (10K block sample count comparison)
   - Phase 3: Transaction spot-checks (3 txs verified against Portal log data)

## Key decisions

- **Start block 17,000,000** — May 2023, after Shanghai upgrade enabled ETH withdrawals. This is when stETH/ETH peg dynamics became most interesting.
- **Table name**: `lido_token_rebased` (auto-generated by CLI from contract name + event name in snake_case).
- **CollapsingMergeTree** engine for reorg handling with `sign` column.
- **UInt256** for all amount fields to preserve full precision of on-chain values.
- **Exchange rate**: Computed as `postTotalEther / postTotalShares` — this gives the ETH value of 1 stETH share.

## Issues encountered

- **Proxy ABI**: The Lido contract uses Aragon's AppProxyUpgradeable pattern, not EIP-1967. Both the CLI and `evm-typegen --chain-id 1` only fetched the proxy ABI with `ProxyDeposit` event. Resolved by manually writing the `TokenRebased` event definition with the correct keccak256 topic0 hash.
- **TokenRebased topic0**: Computed as `keccak256('TokenRebased(uint256,uint256,uint256,uint256,uint256,uint256,uint256)')` = `0xff08c3ef606d198e316ef5b822193c489965899eb4e3c248cea1a4626c3eda50`.
