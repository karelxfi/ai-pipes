# Output — Aave V3 Liquidation Cascades

## What was done

1. **Scaffolded indexer** using `@iankressin/pipes-cli` with the `custom` template, targeting the Aave V3 Pool contract on Ethereum mainnet (`0x87870Bca3F3fD6335C3F4ce8392D69350B4fA4E2`) from block 18,000,000 (~Sep 2023).

2. **Fixed proxy contract ABI** — The CLI fetched the proxy ABI (only `Upgraded` event). Regenerated types from the implementation contract (`0x5FAab9E1adbddaD0a08734BE8a52185Fd6558E14`) using `@subsquid/evm-typegen` and updated the import in `src/index.ts`.

3. **Configured dedicated ClickHouse database** — Changed from default `pipes` to `aave_v3_pool` to avoid sync table conflicts with other indexers.

4. **Created dashboard** — Four-chart layout: liquidation volume over time, top collateral assets, daily liquidation count, and top liquidators. Uses TradingView Lightweight Charts, queries ClickHouse directly.

5. **Created validation script** — Checks table existence, row count, schema columns, timestamp range, and non-negative amounts.

## Key decisions

- **Start block 18,000,000** — September 2023, provides good coverage of recent market stress events including the March 2024 volatility.
- **Table name**: `aave_v_3_pool_liquidation_call` (auto-generated by CLI from contract name + event name in snake_case).
- **CollapsingMergeTree** engine for reorg handling with `sign` column.
- **UInt256** for `debtToCover` and `liquidatedCollateralAmount` to preserve full precision of on-chain amounts.

## Issues encountered

- **Proxy ABI**: As expected, the Aave V3 Pool is a proxy contract. The CLI fetched only the proxy ABI with `Upgraded` event. Fixed by running `evm-typegen` against the implementation address.
- **Node.js v25**: The dev environment runs Node v25.8.1 which has known zstd issues for large syncs. Should work for testing but may need Node 22 LTS for full historical sync.
